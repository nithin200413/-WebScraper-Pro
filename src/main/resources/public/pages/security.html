Security

How we protect your account

Password protection

Passwords are hashed using BCrypt with a per-user salt and a work factor of 12. We never store or log plain-text passwords.

Database access

All database queries use parameterized prepared statements to prevent SQL injection. Connections are pooled and managed with HikariCP.

Sessions

Authentication uses server-side sessions. Your session is invalidated when you sign out.

Reporting a vulnerability

Found a security issue? Please report it responsibly through our Contact page and allow time for a fix before public disclosure.

Good practices

  • Use a unique, strong password.
  • Sign out on shared devices.
  • Only scrape sites you are permitted to access.
